# TradeAgent

Working product name: Parallax. Folder name kept as `TradeAgent` for continuity with the original request.

Working folder for iterating on a financial/trading analysis agent harness with a council of multiaxial paradigms and expert personas.

The current best solution is not a "trading bot with many opinions." It is a governed analysis harness:

1. deterministic market, portfolio, backtest, and risk tools are the source of numerical truth;
2. LLM/persona agents operate as structured analysts, skeptics, explainers, and reviewers;
3. a council protocol forces independent analysis, adversarial critique, synthesis, and veto checks;
4. a thesis lifecycle engine keeps each conclusion temporary, monitored, expiring, and revalidated as markets change;
5. live execution is outside the default system and can only be reached through explicit risk, compliance, and human approval gates.

This folder contains:

- [SPEC.md](SPEC.md): canonical product and implementation specification for Parallax.
- [PHASED_IMPLEMENTATION_PLAN.md](PHASED_IMPLEMENTATION_PLAN.md): high-level roadmap from foundation through permissioned execution.
- [phases/](phases): detailed implementation plan for each phase.
- [iteration_log.md](iteration_log.md): 100 iterations from seed A, 20 optimization iterations, plus a second-seed convergence run.
- [best_solution.md](best_solution.md): the converged architecture, interfaces, council design, and operating model.
- [acceptance_criteria.md](acceptance_criteria.md): why this version satisfies the "I do not see a better solution" stopping condition.
- [sources.md](sources.md): current regulatory and risk-management anchors used to keep the design grounded.

This is an architecture/design artifact, not investment advice and not a recommendation to trade any instrument.
